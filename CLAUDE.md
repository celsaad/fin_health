# Project Instructions

## Quality Standards

- All implementations must pass lint and tests before being considered complete.
- Run the relevant linter and test suite after making changes to verify nothing is broken.

## New Features

- Every new feature must include corresponding tests.
- Ensure accessibility (a11y) considerations are addressed (semantic HTML, ARIA attributes, keyboard navigation, screen reader support).
- Ensure internationalization (i18n) considerations are addressed (all user-facing strings should use the i18n system, no hardcoded text).
